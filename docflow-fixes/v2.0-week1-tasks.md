# DocFlow 2.0 — Týden 1 Tasks (Capacitor Setup)

**Cíl týdne:** Mít funkční Capacitor projekt s prázdným Android buildem, který se nainstaluje na váš telefon a otevře vaši stávající webovou apku.

**Doba:** ~2-3 hodiny denně, 5 dní

---

## 🛡️ DŮLEŽITÉ — bezpečnost před začátkem

**NESAHEJTE na `C:\banbosh\smlouvy-app`!** Tento projekt je nasazený na Firebase a běží mu Closed Test. Capacitor projekt postavíme **v nové složce** `C:\banbosh\docflow-v2`.

---

## DEN 1 (~30 min) — Příprava projektu

### 1) Vytvořit novou složku pro v2.0
```powershell
mkdir C:\banbosh\docflow-v2
cd C:\banbosh\docflow-v2
```

### 2) Init npm projekt
```powershell
npm init -y
```

### 3) Install Capacitor core + CLI + Android
```powershell
npm install @capacitor/core @capacitor/cli @capacitor/android
```

⏳ Trvá ~30-60 s.

### 4) Init Capacitor
```powershell
npx cap init "DocFlow" "cz.banbosh.docflow" --web-dir="public"
```

Vytvoří `capacitor.config.ts`.

### 5) Vytvořit `public/` složku a zkopírovat web
```powershell
mkdir public
Copy-Item C:\banbosh\smlouvy-app\* public\ -Recurse -Force -Exclude functions,android-twa,firebase.json,firestore.rules
```

### 6) Ověřit
```powershell
ls public
```
Měl by tam být `index.html`, `qrcode.min.js`, `sw.js`, ikony, atd.

**📨 Pošlete mi výstup posledního příkazu (`ls public`).**

## DEN 2 (~30 min) — Konfigurace

### 1) Otevřít `capacitor.config.ts` a nahradit obsah
*(Já vám pak pošlu přesný obsah)*

### 2) Přidat Android platform
```powershell
npx cap add android
```

⏳ Trvá ~1 min, stáhne Gradle wrapper.

### 3) Ověřit, že vznikla složka `android/`
```powershell
ls android
```

Měli byste vidět: `app`, `build.gradle`, `gradle`, `gradlew.bat`, `settings.gradle`, ...

**📨 Pošlete mi výstup `ls android`.**

---

## DEN 3 (~45 min) — Sync + první build

### 1) Sync web do native
```powershell
npx cap sync android
```

### 2) Otevřít projekt v Android Studio
```powershell
npx cap open android
```

Otevře se Android Studio, načte projekt, **Gradle sync** běží 2-5 min (jednorázově).

### 3) Připojit telefon přes USB
- V telefonu: Settings → System → Developer options → **USB debugging ON**
- Na PC: telefon se objeví v Android Studio nahoře (název zařízení)

### 4) Run app
V Android Studio nahoře klik **▶ Run 'app'**

⏳ Build trvá 2-5 min poprvé. Pak se apka **nainstaluje** a **spustí** na telefonu.

**📨 Pošlete mi:**
- Screenshot Android Studio s viditelným zařízením
- Screenshot apky na telefonu (jak vypadá)
- Případnou chybu, pokud build selže

## ❓ FAQ

### „Apka se neotevře / vidím bílou obrazovku"
Pravděpodobně problém s bundlingem `public/`. Pošlete mi:
- Screenshot Android Studio Logcat (filter: `Capacitor`)

### „Build padá s gradle chybou"
Pošlete mi:
- Plný text chyby z Android Studio
- `gradlew.bat --version` z `android/` složky

### „Telefon se nezobrazuje v Android Studio"
- Zkontrolujte USB debugging v telefonu
- Vyzkoušejte jiný USB kabel
- V Android Studio: Tools → Device Manager → Pair using USB
